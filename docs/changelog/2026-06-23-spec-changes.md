## 2. 種目ブランドとロゴの統一

- ブランド名を確定：**golf=「ゴルフとも」／running=「ランニングとも」／outdoor=「アウトドアとも」**。
- 3 種目とも **緑（`#1b6e3b`）の線画 SVG ロゴ**に統一（スウッシュ＋種目モチーフ：golf=フラッグ＋ボール、
  running=ランナー＋道、outdoor=山＋太陽）。
  - 総合トップの種目カード（`apps/web/public/{golf,running,outdoor}-logo.svg`）。
  - 各種目アプリのヘッダー（golf は写真調 png を廃止し SVG へ。running/outdoor はヘッダーに
    ロゴ＋ブランド名を新規表示、`appName` は空に）。
- 画像最適化：golf ヘッダー/ヒーローの大きな png を整理（golf ヒーロー 1.48MB→124KB WebP、未使用 jpg 削除）。

## 3. 共通ヘッダーの統一（`packages/shared-ui/SiteHeader`）

- 総合トップのヘッダーを種目アプリと統一：
  - **未ログイン → 「ログイン」**（account 共通ログインへ。戻り先付き）。
  - **ログイン済み → 「プロフィール」**（共通アカウント `account/profile` へ。編集後は総合トップへ戻す）。
    総合に個人マイページが無いため「マイページ」表記は使わない（profile に飛んで実体不一致になるため）。
- `SiteHeader` に **`accountLabel`**（既定「アカウント」）を追加。総合は「プロフィール」を渡す。
  account 等の他アプリは既定のまま変更なし。
- ロゴ横の重複表記を解消（総合の `appName` を空に）。

## 5. フッターの左右バランス配置

- フッターを flex 化：**PC は横並び（`justify-between`）／モバイルは縦積み**。

## 運用メモ（デプロイ）

- **デプロイは 1 サービスずつ直列**で実行する。前の `docker compose up -d --build` 完了前に別の
  `up --build` を重ねると**コンテナ名衝突**（`The container name "/spotomo-…-1" is already in use`）で
  中断し、caddy 以外が「Created（未起動）」のまま全停止する事故が発生した。
- 復旧手順：`docker compose down --remove-orphans && docker compose up -d`（既存イメージから再起動）。
- 詳細は memory `deploy-serial-no-concurrent-compose` に記録。
