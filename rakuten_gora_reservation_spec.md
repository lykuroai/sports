# ゴルフ場予約連携仕様書（楽天GORA API）

## 1. 目的

本仕様は、スポーツ・レジャー仲間募集システムにおける **ゴルフ場検索・ゴルフ場予約導線** の仕様を定義する。

ゴルフ場検索および予約導線については、**楽天GORA API** を利用する。

本システムでは、楽天GORA APIから以下の情報を取得し、利用者に表示する。

- ゴルフ場検索
- ゴルフ場詳細
- プレープラン検索
- 空き枠状況
- 料金
- スタート時間帯
- 昼食付き、キャディ付き、カート有無などのプラン条件
- 予約ページURL

ただし、MVPでは本システム内でゴルフ場予約を直接確定しない。

予約確定は、楽天GORAの予約ページへ遷移して行う。

---

## 2. 基本方針

### 2.1 本システムの役割

本システムは以下を担当する。

- 楽天GORA APIからゴルフ場・プラン情報を取得する
- 利用者にゴルフ場・プラン候補を表示する
- 主催者が選択したプランを仲間募集に紐づける
- 参加者募集、参加申請、承認、チャット、通知を管理する
- 楽天GORA予約ページへの導線を提供する

### 2.2 楽天GORA側の役割

楽天GORA側で以下を行う。

- ゴルフ場予約の確定
- 予約内容の変更
- 予約キャンセル
- 予約に関する決済・ポイント・楽天会員処理
- ゴルフ場側との予約管理

### 2.3 MVPで行わないこと

MVPでは以下を実装しない。

- 本システム内でのゴルフ場予約確定
- 楽天GORA予約の自動実行
- 楽天会員ログインとの直接連携
- 予約変更・キャンセルの自動処理
- 楽天GORA予約履歴の自動同期
- 本システム内でのゴルフ場決済

## 3. 利用する楽天GORA API

| 用途 | API名 | 主な用途 |
|---|---|---|
| ゴルフ場検索 | GORA Golf Course Search API | キーワード、エリア、位置情報からゴルフ場を検索する |
| ゴルフ場詳細取得 | GORA Golf Course Detail API | ゴルフ場IDからゴルフ場詳細情報を取得する |
| プラン・空き枠検索 | GORA Golf Plan Search API | プラン、料金、空き枠、予約ページURLを取得する |

使用バージョンは、楽天Web Serviceで公開されている最新のGORA APIを使用する。

古いAPIバージョンは廃止される可能性があるため、実装時に楽天Web Service公式ドキュメントを確認する。

## 4. ゴルフ場予約の利用フロー

## 4.1 主催者がゴルフ募集を作成する流れ

1. 主催者がログインする
2. 募集作成画面を開く
3. スポーツ種別で「ゴルフ」を選択する
4. プレー希望日、地域、人数、料金条件を入力する
5. 楽天GORA APIでゴルフ場・プランを検索する
6. ゴルフ場・プラン一覧を表示する
7. 主催者が候補プランを選択する
8. 募集人数、最小催行人数、参加条件、説明文を入力する
9. ゴルフ募集を公開する
10. 参加希望者が参加申請する
11. 主催者が参加者を承認する
12. 人数が集まったら、主催者が楽天GORA予約ページへ遷移する
13. 楽天GORA側で予約を確定する
14. 主催者が本システム上で予約状態を「予約済み」に変更する
15. グループチャットで集合時間、予約内容、注意事項を共有する

---

## 4.2 参加者がゴルフ募集に参加する流れ

1. 利用者が「ゴルフ」の募集を検索する
2. 開催日、地域、料金、募集人数を確認する
3. ゴルフ場・プラン情報を確認する
4. 参加条件を確認する
5. 参加申請を行う
6. 主催者が参加を承認する
7. グループチャットに参加する
8. 予約確定後の詳細案内を受け取る
9. 当日参加する
10. 終了後に主催者・参加者を評価する

## 8. 予約状態

ゴルフ募集では、以下の予約状態を管理する。

| 状態コード | 表示名 | 説明 |
|---|---|---|
| `not_reserved` | 未予約 | まだ楽天GORAで予約していない |
| `planning` | 募集中 | 参加者募集中・予約前 |
| `reserved_external` | 楽天GORAで予約済み | 主催者が楽天GORA側で予約済み |
| `changed_external` | 楽天GORAで変更済み | 楽天GORA側で予約内容を変更済み |
| `cancelled_external` | 楽天GORAでキャンセル済み | 楽天GORA側で予約をキャンセル済み |
| `unknown` | 未確認 | 予約状態が確認できない |

MVPでは楽天GORA側の予約状態を自動同期しない。

主催者が手動で予約状態を更新する。

## 10. API設計案

## 10.1 楽天GORA連携API

フロントエンドから楽天APIを直接呼ばず、バックエンド側で楽天GORA APIを呼び出す。

| API | メソッド | 用途 |
|---|---|---|
| `/api/golf/courses` | GET | ゴルフ場検索 |
| `/api/golf/courses/{golfCourseId}` | GET | ゴルフ場詳細取得 |
| `/api/golf/plans` | GET | ゴルフプラン検索 |
| `/api/golf/plans/cache` | POST | プラン情報の一時保存 |
| `/api/recruitments/{id}/golf-plan` | POST | 募集にゴルフプランを紐づける |
| `/api/recruitments/{id}/golf-reservation-status` | PUT | 予約状態を更新する |

---

## 10.2 APIキーをサーバー側で管理する理由

楽天APIキーはフロントエンドに公開しない。

理由は以下のとおり。

- applicationIdやaccessKeyの漏洩防止
- レート制限対策
- レスポンス形式の統一
- キャッシュ制御
- エラー処理の共通化
- 将来のAPI差し替えに対応しやすくするため

---

## 11. 環境変数

```env
RAKUTEN_APPLICATION_ID=
RAKUTEN_ACCESS_KEY=
RAKUTEN_AFFILIATE_ID=
RAKUTEN_GORA_API_BASE_URL=https://openapi.rakuten.co.jp/engine/api
```

APIキーはサーバー側の環境変数として保持し、フロントエンドに公開しない。

---

## 12. キャッシュ方針

楽天GORA APIのレスポンスは、必要に応じて短時間キャッシュする。

| データ | 推奨キャッシュ期間 |
|---|---|
| ゴルフ場基本情報 | 1日 |
| ゴルフ場詳細 | 1日 |
| プラン検索結果 | 15分〜1時間 |
| 空き枠状況 | 5分〜15分 |

料金、空き枠、キャンセル条件は変動する可能性があるため、以下のタイミングで再取得する。

- 募集作成時
- 募集編集時
- 予約ページ遷移前
- 開催日が近い募集を表示するとき

## 14. 表示上の注意

楽天GORA由来の情報を表示する場合、以下を明示する。

- 情報提供元が楽天GORAであること
- 料金や空き枠は最新でない可能性があること
- 予約確定は楽天GORA側で行うこと
- キャンセル規定は楽天GORAおよびゴルフ場側の条件に従うこと

表示例：

> このゴルフ場・プラン情報は楽天GORAから取得しています。料金・空き枠は変更される場合があります。予約確定は楽天GORAの予約ページで行ってください。

## 15. 法務・利用規約上の注意

楽天GORA APIを利用する場合、楽天Web Serviceの利用規約、API利用条件、アフィリエイト条件、表示義務を遵守する。

特に以下を確認する。

- API利用規約
- データの保存可否
- キャッシュ期間
- 表示義務
- アフィリエイトリンク利用条件
- ロゴ・商標の利用条件
- 予約ページへの導線条件

本仕様では、楽天GORAの予約を代行するのではなく、楽天GORAの予約ページへ送客する設計とする。

## 16. 既存システムとの関係

ゴルフのみ以下の特別扱いとする。

1. ゴルフ施設は通常の `facilities` にも登録する
2. 楽天GORA連携情報は `golf_courses` に保持する
3. ゴルフ募集は通常の `recruitments` に登録する
4. ゴルフ固有情報は `golf_plans` と `recruitment_golf_details` に保持する
5. ゴルフ場予約確定は楽天GORA側で行う
6. 本システムは仲間募集、参加管理、チャット、通知を担当する

## 18. Claude Code向け実装指示要約

ゴルフ場予約部分は、楽天GORA APIを利用して実装する。

- フロントエンドから楽天APIを直接呼ばない
- サーバー側APIで楽天GORA APIを呼び出す
- APIキーは環境変数で管理する
- ゴルフ場検索、詳細取得、プラン検索を実装する
- 予約確定は楽天GORA予約ページへ遷移して行う
- 本システム内では予約を確定しない
- 選択したゴルフ場・プランを募集に紐づける
- 主催者が楽天GORA側で予約後、予約状態を手動更新できるようにする
- 参加者管理、チャット、通知は本システム側で行う
